# Прогнозирование откликов соискателей на вакансии (Avito Работа)

## Описание задачи

На платформе Авито Работа необходимо предсказать вероятность контакта соискателя с работодателем для каждой пары (поисковый запрос, вакансия).  

- **Целевая переменная:** `item_contact` (1 — контакт был, 0 — не было).  
- **Метрика:** ROC-AUC, оценивает способность модели различать пары с контактом и без.  
- **Формат ответа:** CSV с колонками `query_id`, `item_id`, `item_contact` (вероятность контакта).

---

## Подход к решению

1. **Загрузка данных**
    - Данные представлены в формате `.parquet`.
    - Разделены на `train` и `test`.

2. **Feature Engineering**
    - **Сравнение категорий и локаций:** совпадение категории вакансии и запроса (`same_cat`, `same_mcat`), совпадение локаций (`same_loc`).  
    - **Текстовые признаки:** длина текста запроса, заголовка и описания; overlap между запросом и заголовком вакансии.  
    - **Цена:** логарифм зарплаты `price_log` и индикатор отсутствия значения `price_is_null`.  
    - **Время:** час и день недели события.  
    - **Исторические признаки:** средняя конверсия `seller_ctr`, `item_ctr` для каждого работодателя и вакансии.  
    - Обработка пропусков и выбросов для корректной работы модели.

3. **Подготовка категориальных признаков**
    - Категориальные признаки (`serp_query_cat`, `query_mcat`, `item_cat_id`, `item_mcat_id`, `serp_query_loc`, `item_loc`) были приведены к строковому типу для корректной работы CatBoost.

4. **Модель**
    - Использована **CatBoostClassifier**.
    - Параметры: `iterations=500`, `depth=6`, `learning_rate=0.07`.
    - Обучение с использованием `GroupKFold` для предотвращения утечки по `query_id`.

5. **Предсказание**
    - Используем `predict_proba` и берём колонку класса 1 для вероятности контакта.  
    - Полученные значения сохраняем в файл формата CSV
